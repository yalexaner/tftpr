# Add cargo-dist and cargo-release for automated releases

## Overview

Set up cargo-dist for automated cross-platform binary builds and GitHub releases, and cargo-release for streamlined version bumping and tag management. The workflow: run cargo-release to bump version and push a tag, which triggers cargo-dist's GitHub Actions CI to build binaries for all target platforms and publish a GitHub release.

## Context

- Files involved: Cargo.toml, .github/workflows/release.yml (generated by cargo-dist), release.toml (cargo-release config)
- Related patterns: none (no existing CI/CD)
- Dependencies: cargo-dist (installed via cargo), cargo-release (installed via cargo)

## Development Approach

- No tests needed for CI/CD tooling configuration
- Validate by doing a dry-run release
- Complete each task fully before moving to the next

## Implementation Steps

### Task 1: Add required Cargo.toml metadata

**Files:**
- Modify: `Cargo.toml`

cargo-dist and GitHub releases require additional package metadata (description, license, repository, homepage).

- [x] Add `description`, `license`, `repository` fields to [package] in Cargo.toml
- [x] Add a LICENSE file if one doesn't exist (matching the license field)

### Task 2: Initialize cargo-dist

**Files:**
- Modify: `Cargo.toml` (cargo-dist adds [profile.dist] and [workspace.metadata.dist])
- Create: `.github/workflows/release.yml` (auto-generated by cargo-dist init)

- [x] Run `cargo dist init` to generate the release CI workflow and dist configuration
- [x] Configure target platforms: x86_64-unknown-linux-gnu, aarch64-unknown-linux-gnu, x86_64-apple-darwin, aarch64-apple-darwin, x86_64-pc-windows-msvc
- [x] Review generated .github/workflows/release.yml and Cargo.toml changes
- [x] Run `cargo dist plan` to verify the build plan lists all expected targets

### Task 3: Configure cargo-release

**Files:**
- Create: `release.toml`

- [x] Create release.toml with configuration: tag-prefix = "", push-remote = "origin", tag-message = "{{version}}"
- [x] Add pre-release-replacements if needed (e.g., CHANGELOG references)
- [x] Run `cargo release version patch --dry-run` to verify configuration works

### Task 4: Verify end-to-end workflow

- [x] Run `cargo dist plan` to confirm all 5 target triples appear
- [x] Run `cargo release version patch --dry-run` to verify version bump
- [x] Run `cargo dist build` locally to verify the current platform binary builds correctly
- [x] Document the release process in README.md: `cargo release patch` (or minor/major) triggers the full pipeline
